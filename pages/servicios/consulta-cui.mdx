# Consulta de CUI (DPI)

Web Service para consultar el **nombre registrado** asociado a un CUI/DPI, para agregarlo a sus Documentos Tributarios Electrónicos previo a la certificación.

## 1. Introducción

Este manual describe el uso del Web Service destinado a la consulta de CUI (DPI) para agregarlo a sus Documentos Tributarios Electrónicos previo a la certificación. El servicio retorna el nombre registrado del CUI consultado mediante la API de INFILE.

## 2. Requisitos Previos y Restricciones

- Credenciales válidas y activas para consumir el servicio (Usuario API, Llave API).
- Conocimiento básico en consumo de servicios web (REST).
- Capacidad de generar el JSON/form-data de autenticación y consulta.
- El CUI a consultar debe ser válido y no debe mostrar un estatus de fallecido en los registros.
- El servicio **solamente retorna el nombre registrado** del CUI asociado.

## 3. Endpoints

Este servicio requiere **dos llamadas**: primero autenticarse para obtener un JWT, luego usar ese JWT para realizar la consulta.

### 3.1 Autenticación

**Método:** `POST`

```
https://certificador.feel.com.gt/api/v2/servicios/externos/login
Content-Type: multipart/form-data
```

### 3.2 Consulta de CUI

**Método:** `POST`

```
https://certificador.feel.com.gt/api/v2/servicios/externos/cui
Content-Type: multipart/form-data
Authorization: Bearer {JWT}
```

## 4. Proceso de Consulta

<Steps>

### Paso 1 — Autenticación (obtener JWT)

El primer paso es autenticarse para obtener un **JWT (JSON Web Token)** válido por **2 horas**.

Enviar un `form-data` con las credenciales:

| Campo | Descripción |
|---|---|
| `prefijo` | Usuario API |
| `llave` | Llave API |

Ejemplo con `curl`:

```bash
curl -X POST https://certificador.feel.com.gt/api/v2/servicios/externos/login \
  -F "prefijo=tu_usuario_api" \
  -F "llave=tu_llave_api"
```

### Paso 2 — Respuesta de autenticación

El servicio retorna un JSON con el JWT válido por 2 horas:

```json
{
  "resultado": true,
  "descripcion": "Autenticación exitosa.",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiJ0dV91c3VhcmlvX2FwaSIsImlhdCI6MTYwMDAwMDAwMCwiZXhwIjoxNjAwMDA3MjAwfQ.FIRMA_DEL_TOKEN"
}
```

<Callout type="warning">
  El JWT tiene una validez de **2 horas**. Pasado ese tiempo debes generar un nuevo token repitiendo el Paso 1.
</Callout>

### Paso 3 — Consulta del CUI

Con el JWT obtenido, enviar la consulta usando el token como `Bearer Token` en el header de autorización y el CUI en el `form-data`.

```bash
curl -X POST https://certificador.feel.com.gt/api/v2/servicios/externos/cui \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -F "cui=3256789012345"
```

| Campo (form-data) | Descripción |
|---|---|
| `cui` | Número de CUI/DPI a consultar (13 dígitos) |

### Paso 4 — Respuesta de la consulta

El sistema retorna un JSON con el estado y el nombre de la persona asociada al CUI:

```json
{
  "resultado": true,
  "nombre": "JUAN ANTONIO GARCIA LOPEZ",
  "estatus": "ACTIVO"
}
```

</Steps>

## 5. Manejo de Errores

### JWT expirado o inválido

```json
{
  "resultado": false,
  "descripcion": "Token inválido o expirado. Por favor autentíquese nuevamente."
}
```

### CUI no encontrado o con estatus fallecido

```json
{
  "resultado": false,
  "nombre": null,
  "estatus": "NO_ENCONTRADO",
  "descripcion": "El CUI ingresado no se encuentra en los registros."
}
```

<Callout type="info">
  Se recomienda implementar un mecanismo de **renovación automática del JWT** en tu sistema, para detectar cuando el token está próximo a expirar (2 horas) y autenticarse nuevamente sin interrumpir el flujo de emisión de documentos.
</Callout>
